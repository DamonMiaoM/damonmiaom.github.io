不久前，我读到一篇关于 Boris Cherny 的访谈——Claude Code 的创造者，他在 AI Ascent 2026 上分享了自己的工作方式：几百个 Agent 同时在跑，核心是他自创的「Sloop」——让 Claude 用 Cron 调度重复任务，形成持续运转的**循环（loop）**。

我当时读完，隐约觉得这个词有什么东西，但没有完全抓住。

今天，我又读到一篇介绍 Dexter 的文章——一个专为金融研究设计的 AI Agent，GitHub 上 23.5K Star。文章反复出现同一个词：**loop**。在 AI 的帮助下我才知道，loop 在这里指的就是**迭代**——Agent 不断调用 LLM、执行工具、管理上下文、再调用 LLM。

这一次我没有放过它。这个循环背后，有没有一个更本质的数学结构？

## 一个公式就够了

我和 AI 把 Agent 循环抽象成了一个递推公式：

$$x_{t+1} = a \cdot x_t + b$$

*（严格来说，真实 LLM 的状态是高维向量，a 是一个算子而非标量。这里用标量建立直觉——足够抓住核心结构，但不要把它当成精确模型。）*

其中：
- $x_t$ 是当前状态（上下文、记忆、已有推理）
- a 是 LLM 对现有状态的变换（理解、压缩、推理）
- b 是外部新输入（工具返回、用户追问、新数据）

每一轮循环，agent 的状态 = 对已有认知的加工 + 新信息。

这个公式虽然简单，但已经能解释 Dexter 设计中的大部分工程决策：

- microcompact（微压缩）就是在控制 a，定期衰减旧信息
- tool 调用就是在注入 b，带来新的外部数据
- 上下文管理就是在确保 $a < 1$，让系统收敛而非发散

## 不动点：系统想去的地方

公式 $x_{t+1} = a \cdot x_t + b$ 有一个不动点：

$$x^* = \dfrac{b}{1-a}$$

不动点就是系统反复迭代后最终停留的位置。当你令 $x_{t+1} = x_t$，代入求解，就得到这个值。

它的物理意义很清楚：**一旦到达，就不再变化。** 代回去等式成立，没有变化的动力了。就像一个球滚到碗底——不管你从碗的哪一边放下去，最终都停在最低点。

但关键问题是：系统真的会到达不动点吗？

## 收敛与发散

偏差 $e_t$ = $x_t$ - $x^*$ 本身也遵循同样的递推关系：

$$e_{t+1} = a \cdot e_t$$

偏差每轮乘以 a。所以：

- $|a| < 1$ → 偏差每轮缩小，越来越接近 $x^*$，**收敛**
- $|a| > 1$ → 偏差每轮放大，离 $x^*$ 越来越远，**发散**
- $|a| = 1$ → 偏差不变，永远到不了 $x^*$

用具体数字感受一下。设 $a = 0.5, b = 1$，不动点 $x^* = 2$：

```plain
t=0  x=0    e=-2
t=1  x=1    e=-1      （偏差缩小一半）
t=2  x=1.5  e=-0.5
t=3  x=1.75 e=-0.25
```

越来越接近 2。收敛的本质是：**偏差以 a 的速率衰减。**

映射回 agent，a 就是「压缩率」——你每轮保留多少旧信息。压缩太轻（a 接近 1），收敛慢；压缩太重（a 接近 0），记忆丢失太快。Dexter 的工程选择本质上就是：确保 $a < 1$，让系统有不动点，能收敛。

## a 和 b：两种纠正机制

到这里，我想到了头马演讲俱乐部的训练模式。

你上台演讲（$x_t$），evaluator 给你反馈，你的下一次演讲（$x_{t+1}$）取决于你怎么整合这个反馈。evaluator 就是 b。

但 b 的质量决定了方向：

```plain
高质量 b（好 evaluator）→ 纠正偏差，逼近不动点
低质量 b（烂 evaluator）→ 注入垃圾，放大偏差
```

Garbage in, garbage out。b 不是天然好的，它只是一扇门——好东西和垃圾都能进来。

那 a 里有没有纠正？有。a 是 LLM 对现有状态的变换，这个变换不是机械复制，而是理解、压缩、筛选。一个好的模型在 a 这一步会过滤掉上一轮的噪声，提炼出关键信息，修正推理中的逻辑跳跃。所以 a 本身也包含纠正——它是**内部纠正**，靠模型自身的推理能力。

对应到头马：

```plain
a = 你自己的反思能力（内部纠正）
b = evaluator 的反馈（外部纠正）
```

只有 a 没有 b → 闭门造车，自我感觉良好但实际在原地转。只有 b 没有 a → 听了一堆反馈但不会消化，越改越乱。

回到不动点 $x^* = b/(1-a)$：a 决定了你的学习效率（消化反馈的速度），b 决定了你的学习方向（反馈的质量）。两者缺一不可。

而最危险的情况是：$a > 1$（自大，放大自己的偏见）+ b 质量差（烂 evaluator 吹捧你）。这时候系统不仅不收敛，还会加速冲向悬崖，而且你还觉得自己在进步。

## 开放系统意味着什么

但这里有一个更深的问题：什么叫「接受反馈」？

头马的设计里，有一个细节经常被忽略——evaluator 本身也被评估。每次会议有一个 General Evaluator，专门评价「evaluator 的评价质量如何」。这是一个**对反馈的反馈**，meta-feedback。它保证的不是当下的 b 质量，而是 b 的质量会随时间提升。

对应到公式：不只是 $x_{t+1} = a \cdot x_t + b$，而是 b 本身也在迭代——$b_{t+1}$ 会比 $b_t$ 更精准。整个系统在两个维度上同时收敛：你的表现在收敛，给你反馈的人也在收敛。

这就是一个**开放系统**的完整图景：

```plain
开放 ≠ 被动接收一切输入
开放 = 主动寻求高质量输入 + 有能力区分信号与噪声 + 对输入保持结构性的响应
```

头马训练的不只是演讲技巧，它训练的是一种**接收反馈的身体习惯**——你站在台上，有人当面告诉你哪里不好，你要做的不是辩解，而是听完、消化、下一次改。这个过程，本质上是在训练 $a < 1$：让自己的固有认知留出空间，让外部信息真正进入并改变你。

一个封闭系统的特征是 $a \geq 1$：无论你给它什么输入，它都用自己原有的框架去消化，输出强化自身偏见的结果。你跟它说什么，它都能解释成「我是对的」。这不是在接收 b，而是在消灭 b。

而一个开放系统的特征是 $a < 1$：输入真的能改变状态。你听完 evaluator 的话，下次演讲真的不同了。不是因为你在「听话」，而是因为你的内部结构被那个反馈重塑了。

映射回 agent 设计：Dexter 的 web_search 三级降级链（Exa → Perplexity → Tavily）就是在保证 b 的质量——如果最好的数据源不可用，降级到次好的，而不是拿到垃圾数据。而它的 validation 层（检查自己的答案是否自洽）就是在确保 $a < 1$——让推理结果真的被外部证据修正，而不是自我强化。

头马的智慧可以浓缩成一句话：**成长不是来自你知道多少，而是来自你愿意被什么改变。** 这句话对人成立，对 agent 也成立。

## 烂反馈的催化价值

但这里还有一个反直觉的发现：b 的质量差，不代表 b 不能催生好东西。

前几天听老罗和蔡康永的对话，康永哥提到读烂书的意义。他说，读到烂书表达某个观点的时候，他可能会惊叹：这个观点怎么能用如此糟糕的方式表达？而他转念会想：如果让我来说，我会怎么说？

这个「editing」的冲动，反而促使他找到了更精致的表达。

这彻底改变了我们对 b 的理解。之前说「garbage in, garbage out」——那是把 b 当作被动的输入。但实际上，b 还可以是**催化剂**。催化剂本身不进入产物，但它激活了反应：

```plain
好 b → 直接改善 x（吸收式学习）
烂 b → 激活 a 的创造性功能（催化式学习）
```

催化式学习产出的东西，可能跟 b 完全不同。你读了一本烂书关于「人际关系」的糟糕表达，最终写出了一段关于「沟通的艺术」的精彩文字。这段文字跟那本烂书几乎没有关系，但没有那本烂书，你也不会写出来。

烂书的价值不在于它说了什么，而在于它逼你自己去找答案。好书给了你答案，你接收就好；烂书差到你不得不自己重构一遍——而重构的过程比接收的结果更深刻。

所以完整的图景是：

```plain
b 的质量高 → x 直接受益（收敛更快）
b 的质量低 → 不一定有害，可能激发出 x 的创造性跳跃
没有 b → 闭门造车，连触发反应的机会都没有
```

最可怕的不是收到烂反馈，而是**没有任何反馈**。一个没有 b 的系统，即使 a 再强，也只能在自己的封闭世界里打转。烂书至少给了你一个「这不对」的起点，而没有书，你连「不对」都无从感知。

所以 garbage in, 不一定是 garbage out。如果 a 足够强，可以 gold out。关键不在于 b 的质量，而在于你有没有一个能够从任何 b 中提取价值的 a。

## 黑洞与吸引子

这个不动点让我想到了黑洞。

动力系统里有一个概念叫**吸引子（attractor）**——系统向某个稳定态演化的结构。不动点就是最简单的一种吸引子。而黑洞，就是物理世界里的吸引子。

对应关系非常精确：

| 数学 | 物理 |
|------|------|
| 不动点 $x^*$ | 奇点（黑洞中心） |
| 收敛区域 | 事件视界 |
| $\|a\| < 1$ | 引力足够强 |
| 从任意起点出发 | 任何落入视界的物质 |

但有一个区别：在我们的线性公式里，$x_t$ 永远趋近但永远到不了 $x^*$，数学上叫「渐近收敛」。而真实黑洞更狠——物质不是无限趋近奇点，是真的到达了。这是非线性效应：越靠近中心，引力越强，a 本身在变大，收敛加速，最终变成撞击。

从常数 a 到函数 $a(x)$，就是从线性跨到非线性。简单公式失效了，但真实世界就是非线性的。

## 熵增：收敛不是默认的

到这里，我意识到之前的讨论有一个隐含的乐观假设：系统会自然收敛到不动点。

但真的是这样吗？

熵增定律说的恰恰相反：**系统默认趋向无序，有序是需要能量维持的例外。** 一杯热水自然冷下去（收敛），但一杯冷水不会自然变热。逆熵需要做功。

$|a| < 1$ 才收敛，$|a| > 1$ 就发散。如果没有主动设计，$|a| > 1$ 容易成为默认状态——LLM 推理本身虽然有一定的内在压缩能力，但在没有上下文管理的裸跑场景下，信息膨胀是大概率事件。

没有上下文压缩机制，agent 的上下文会爆炸。没有 validation 层，错误会在下一轮被放大。没有最大轮数限制，循环可能无限跑下去。

发散有两种形态：

$a > 1$ 时，正向发散——滚雪球。agent 每轮输出越来越长、越来越复杂，上下文越来越满，LLM 看到满屏历史输出更长的回应，正反馈，直到 token 耗尽。

$a < -1$ 时，震荡发散——来回拉扯。agent 在两个极端之间反复，先说「营收增长强劲」，下一轮说「等等，其实是下降的」，再下一轮又翻转，每轮都在推翻上一轮，越来越激进。

**收敛是被设计出来的，发散才是自然状态。** Agent 框架的全部工作，本质上就是在对抗熵增。

## Loop 套 Loop：Sloop 与自维持系统

怎么对抗？工程上的回答是：让 loop 变成 Sloop。

就在我们讨论这些的时候，我读到了另一篇文章——Claude Code 创始人 Boris Cherny 在 AI Ascent 2026 峰会上的分享。他提到了一个叫 Sloop 的东西。

Sloop 的本质是：用定时任务（Cron）让 agent 每隔 N 分钟自动跑一次。一个 Sloop 在盯代码的 CI 状态，报错就自动修；另一个每 30 分钟抓 Twitter 上的用户反馈，自动分类汇总。

这跟我们讨论的 Agent Loop 是什么关系？

Agent Loop 是**单次会话内的循环**——agent 调用 LLM → 执行工具 → 把结果喂回 LLM → 再调用，直到任务完成。这是 $x_{t+1} = a \cdot x_t + b$，每一步依赖上一步。

Sloop 是**跨会话的定时循环**——每隔一段时间启动一个新的 Agent Loop，把上一轮的输出作为新的 b 注入。

结构是：

```plain
Sloop（外层递归：时间维度的重复调度）
  └── 每次触发一个 Agent Loop（内层递归：状态维度的迭代求解）
        └── x_{t+1} = a·x_t + b
```

Sloop 是递归套递归。外层是时间维度的重复调度，内层是单次任务的状态迭代。

当多个 Sloop 同时跑起来——一个盯代码、一个盯测试、一个盯用户反馈——项目就变成了一个**自维持系统**：代码坏了自动修，测试挂了自动跑，用户有反馈自动收集。开发者从「干活的人」变成了「看监控的人」。

这让我想到：如果 loop 变成 Sloop，agent 不只是完成一个任务，而是持续存在于环境中，像一个永不休息的员工。从一次性求解到持续运行——这才是真正的质变。

而下一步自然就是：agent 自己决定该跑哪些 Sloop、什么时候加新的、什么时候停掉旧的。那就是自进化的系统了。

工程能做的事情很多，但它有边界吗？

## 从线性到非线性：当 a 和 b 本身在变化

有的。

到目前为止，a 和 b 都是常数。但真实的 agent 不是这样——每一步的压缩策略和信息获取都依赖于当前认知。

$a(x_t)$：上下文越大，压缩率越激烈。上下文小时几乎不压缩（$a \approx 1$），上下文爆满时猛压（$a \ll 1$）。

$b(x_t)$：当前状态决定下一步会调什么工具、拿到什么数据。agent 看到「营收下降」，下一步可能去查「下降原因」；看到「利润率健康」，下一步可能去查「竞争对手」。

方程变成：

$$x_{t+1} = a(x_t) \cdot x_t + b(x_t)$$

没有解析解了——写不出一个公式说 $x^*$ 等于什么。你解不动点了。

但这恰恰是真实 agent 的状态：下一步做什么，取决于这一步理解了什么。这不是 bug，这是 agent 真正在「思考」的标志。

从常数 a,b 到函数 $a(x)$, $b(x)$，本质上是从线性跨到非线性。简单公式失效了，但真实世界就是非线性的。能做的分析变成数值模拟、局部线性化、Lyapunov 稳定性分析——从「求解公式」变成了「观察行为」。

## 哥德尔：数学无法自证

走到非线性，我忽然意识到一个更深的问题。

我们一直在用数学工具分析 agent。但数学本身能不能被数学分析？换句话说，我们用来分析 agent 的这套数学，它自己是 solid 的吗？

答案是：无法自证。

1931 年，哥德尔证明了：

> 任何足够强的、自洽的数学系统，都无法在自身内部证明自己是自洽的。

这就是不完备定理。数学无法证明数学不会出矛盾。你必须跳出这个系统，用一个更大的系统去证明。但那个更大的系统又无法证明自己……无限后退。

我脑海里浮现出这样的画面：一个巨大的 expanding 的玻璃球（universe）之外，有一个更大的空间（在另一个更大的玻璃球中，这个玻璃球也在 expanding）。每一层玻璃球能看见里面的那层，能验证里面的那层，但看不见自己。一层套一层，永远有更大的玻璃球。

而且 expanding 很关键——每一层都在膨胀，新的不可判定命题在不断涌现。也许不存在最终的那个最外层玻璃球。不是有一个「上帝视角」能看清所有层，而是这个套娃本身就是无限的。

## 从内向外的推测

那么，我能否反过来——从里面推测外面？

如果我在 Level 2，我能看到 Level 1 在我内部。我知道 Level 2 和 Level 1 之间的包含关系。我能否用这个关系来推测 Level 3 对 Level 2 的包含？

这本质上是归纳推理：从已观察到的规律，推断未观察到的结构。

但这里有一个关键假设：**同构性**。只有当每一层的包含关系结构相同时，归纳才有效。

实际上，数学里的坏消息是：不同层级之间的关系往往不是同构的。Level 1（算术）里有真但不可证的命题；Level 2（集合论）能证明 Level 1 的那些命题，但自己又产生新的不可证命题；而且 Level 2 新产生的不可证命题，跟 Level 1 的性质不同。

不是简单的「同样事情再发生一次」，而是每一层的困境形态都不同。

而且最残酷的是：你在自己这一层内，无法确定跨层是否同构。这又回到了哥德尔——判定这件事本身可能需要更高层的视角。

## 佛法的回应：不是解决，是消融

到这里，逻辑走到了尽头。我问 AI：如果用佛法的视角和佛陀的智慧，这个问题该如何破解？

但「破解」这个词本身可能就是错的。

哥德尔的困境预设了一个框架：有一个系统，系统内部有盲区，需要外部来验证。这个框架本身是二元的——里面/外面，主体/客体，已知/未知。佛法不打算在这个框架里找答案。它质疑的是框架本身。

龙树的中观说：一切法无自性（śūnyatā）。系统之所以有「内部不可判定的东西」，是因为你假设系统有一个固定的、自有的边界。但如果你去看，这个边界本身也是因缘和合的——它不是独立存在的实体。

这不是「解决了」哥德尔的问题，而是**消融了**问题的前提。dissolution，不是 solution。

当你认识到「玻璃球」没有自性，它不是一个真的东西，而是一个概念投射——问题就不是「怎么看到外面」，而是「内外之分本身是幻象」。

般若（prajñā）的智慧不是在某一层玻璃球里获得更广的视野，而是看穿玻璃球本身不存在。

这在认识论上是什么意思？当你说「我无法看到高于我的那一层」时，你已经在假设有一个「我」，有一层「高于」，有一个「看」。这三个概念全都是建构。拆掉建构，问题不是被解决了，而是消融了。

禅宗有一个公案可以说明这个区别。僧人问赵州：「如何是祖师西来意？」赵州答：「庭前柏树子。」僧人问的是一个关于终极意义的问题，赵州没有回答这个问题——他把问题消解了。你不需要「看到更高一层」，你只需要回到此刻，回到眼前。柏树就在那里，它不需要被证明。

这种解法有一个诚实的局限：它是体验性的，不是逻辑推导出来的。你不能用思维去「想通」无自性，因为「想」本身就是那个玻璃球内的活动。佛法指向的是禅修实践——通过止观，直接体验到「内外之分」的消融。就像你不能通过阅读游泳教材学会游泳，你必须下水。

## 三条路

回头看这场对话，我们从一个 Agent 项目的 src/ 结构出发，走到了三条不同的路上：

数学说：这个问题在逻辑内无解。足够复杂的系统，自指必有盲区。这不是数学的 bug，是逻辑本身的结构性限制。

工程说：虽然不能完美解决，但可以不断逼近。Dexter 的三层压缩、validation 层、Scratchpad——全是在有限资源下对抗发散的务实策略。不是消除盲区，是管理盲区。

佛法说：问题的根源不在于逻辑不够强，而在于你用了逻辑这个工具去碰一个逻辑之外的问题。数学告诉你边界在哪，佛法告诉你边界本身是什么。不是在玻璃球里找到更好的视野，而是发现玻璃球从来就不存在——庭前柏树子，不需要被证明。

三者不矛盾，只是指向不同。

但我知道这只是一个不动点，不是终点。下一次对话，b 会不同，a 会不同，$x^*$ 也会不同。
